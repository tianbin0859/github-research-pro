# 技术实现文档模板

基于GitHub预研结果撰写技术实现文档的标准模板。

## 使用场景

完成GitHub搜索后，不要立刻编程，必须先写技术实现文档并经用户确认。

## 撰写要点

1. **预研结果必须量化**：Stars、评分、对比表格
2. **方案必须有对比**：至少2-3个方案，明确推荐
3. **风险必须分级**：高/中/低，对应不同策略
4. **Checklist必须分Phase**：便于跟踪进度
5. **必须等用户确认**：文档末尾明确标注"请确认方案"

## 用户确认信号

- "好" / "开始" / "确认" / "同意" → 开始编码
- "直接写" / "不用搜" / "我知道怎么做" → 跳过预研
- 选择方案编号（1/2/3）→ 按选定方案编码
- 提出修改意见 → 修改文档后再确认
